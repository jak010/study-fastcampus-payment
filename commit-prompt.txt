아래 형식을 따르도록 변경 사항을 요약해 커밋 메시지를 작성해줘.
규칙:

제목은 50자 이내, 한글 사용, 마지막에 마침표 ❌

영어 동사 원형으로 시작하는 commit type 사용 (feat, fix, docs, refactor, test, chore 등)

제목과 본문 사이에 한 줄 띄어쓰기

본문에는 변경 이유 + 구체적 변경 내용을 포함

필요 시 하단에 연관 이슈 번호와 브레이킹 체인지 표시

형식 예시:

<type>: <요약 메시지>

무엇을 변경했는지 + 왜 변경했는지

- 상세 변경 사항 1
- 상세 변경 사항 2

Refs: #이슈번호
BREAKING CHANGE: 기존 API 응답 필드 제거


입력 예시:

변경 내용:
- 로그인 실패 시 잘못된 메시지가 노출됨
- 예외 처리 로직 추가
- 서버 로그 추가


출력 예시:

fix: 로그인 실패 메시지 노출 오류 해결

로그인 실패 시 잘못된 안내 문구가 노출되는 문제 수정

- 예외 처리 로직 보완
- 에러 상황 로깅 추가

✨ 추가 옵션 (선택)

프롬프트에 아래 옵션을 추가하면 더 좋아집니다:

- 영어/한국어 선택 가능
- 커밋 타입 추천
- 문체 교정(무조건 명령형)
- 이모지 추가 옵션 허용 여부
